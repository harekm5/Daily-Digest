# Daily Brief — Personal Newsletter

A 6 AM daily newsletter, generated by Claude with web search, delivered to your Gmail. Three sections: AI for PMs, Health & Wellness, and Living Well.

## What it does

Every morning at 6 AM Pacific, GitHub Actions runs a Python script that:

1. Fires three Claude API calls (one per category) with web search enabled
2. Parses the structured JSON each one returns
3. Renders a clean HTML email
4. Sends it to your inbox via Gmail SMTP

Each story includes a headline, 2-3 sentence summary, "why it matters" takeaway, and a link.

## One-shot setup (~10 minutes)

### 1. Gather two credentials

**Gmail app password** — Google requires this for SMTP (regular password won't work):
1. Turn on 2-Step Verification: https://myaccount.google.com/security
2. Go to https://myaccount.google.com/apppasswords
3. Create one named "Daily Newsletter" — copy the 16-character output

**Anthropic API key** — https://console.anthropic.com → Settings → API Keys → Create Key. Estimated cost: ~$2-5/month.

### 2. Install the GitHub CLI (if you don't have it)

```bash
brew install gh           # macOS
# or see https://cli.github.com for other platforms
gh auth login             # one-time login
```

### 3. Run the setup script

```bash
cd daily-newsletter
chmod +x setup.sh
./setup.sh
```

It will:
- Verify prereqs
- Prompt for your 4 credentials (Anthropic key, Gmail address, Gmail app password, recipient)
- Create a private GitHub repo and push the code
- Add all 4 credentials as repo secrets
- Trigger a test newsletter to your inbox right away

After this, it runs daily at 6 AM Pacific automatically.

### Manual setup (if you'd rather not use the script)

<details>

1. Create a private GitHub repo, push this folder to it.
2. Repo → Settings → Secrets and variables → Actions → New repository secret. Add four secrets:

| Secret | Value |
|---|---|
| `ANTHROPIC_API_KEY` | Your Anthropic API key |
| `GMAIL_USER` | Your Gmail address |
| `GMAIL_APP_PASSWORD` | The 16-char Gmail app password |
| `RECIPIENT_EMAIL` | Where the newsletter goes |

3. Repo → Actions tab → "Daily Newsletter" → "Run workflow" → leave "force_run" checked → Run.

## Iterating on the prompts (without sending email)

Use `test_local.py` to run categories locally and see what comes back before committing changes:

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY="sk-ant-..."

# Test one category, see the stories in the terminal
python test_local.py ai_for_pms

# Test all three and open a rendered HTML preview in your browser
python test_local.py all --open

# Save the preview to a specific file
python test_local.py all --html preview.html
```

This skips Gmail entirely — pure prompt iteration loop. Tune `prompts.py`, re-run, check the output, repeat. Once happy, push to deploy.

## Tuning the content

All editorial direction lives in `prompts.py`. To change what shows up:

- **Add/remove a category**: Edit `CATEGORY_PROMPTS` and `SECTION_META` together.
- **Change focus areas**: Edit the `system` prompt for that category — be specific about what you want and what to avoid.
- **Change the writers/sources you trust**: The "Prefer:" and "Avoid:" lists in each prompt directly shape what gets surfaced.
- **Change story count**: Edit the "Exactly 3 stories" line in `OUTPUT_INSTRUCTION`.

After editing, commit and push. The next 6 AM run picks up the changes automatically.

## Tweaking the schedule

In `.github/workflows/daily-newsletter.yml`, the cron lines schedule the run. The Python script gates on actual Pacific time so only the right one sends. To change to a different hour, update both the crons and the `should_run_now()` check in `newsletter_generator.py`.

GitHub Actions cron is best-effort — runs may be delayed by a few minutes during peak hours. Acceptable for a 6 AM newsletter, not acceptable if you needed exact-minute precision.

## Troubleshooting

**"Username and Password not accepted" from Gmail**: You're using your regular password, not an app password. Generate one at the link in step 1.

**Newsletter has empty sections**: A category's API call failed or returned malformed JSON. Check the workflow logs — the error is printed inline. Most common cause: a search query returned no fresh results, in which case re-running usually fixes it.

**Want to forward to multiple emails**: Set `RECIPIENT_EMAIL` to a comma-separated list. Or set up a Gmail filter to forward.

**The 6 AM gate seems off after daylight savings**: It shouldn't be — the script reads `America/Los_Angeles` time directly. If something is wrong, check that GitHub Actions runners have the right tzdata (they do by default on `ubuntu-latest`).
